fgt

fgt is an LGPL C++ library for quickly performing Gauss transforms. fgt can do an Improved Fast Gauss Transform, a direct Gauss transform using an approximate nearest neighbor simplification, and the direct Gauss transform. This library is heavily based on previous work in the figtree library, as well as the original Improved Fast Gauss Transform code.

fgt lives at https://github.com/gadomski/fgt, and its documentation lives at http://gadomski.github.io/fgt.

Usage

Let fgt pick the best Gauss transform for your purposes:

GaussTransformUnqPtr transfom = choose_gauss_transform(source, bandwidth, epsilon);
auto transform = transform->compute(target, weights);

Run the Improved Fast Gauss Transform:

Ifgt ifgt = Ifgt(source, bandwidth, epsilon);
auto transform = ifgt.compute(target, weights);

Dependencies

fgt requires a modern compiler that supports many c++11 features, such as clang 3.4 or gcc 4.8, and armadillo, a C++ linear algebra library. On OSX, installing armadillo is easy with homebrew:

1 $ brew install armadillo

On Ubuntu:

1 $ sudo apt-get install libarmadillo-dev

Windoze, caveat emptor.

Installation

fgt uses CMake:

1 $ mkdir build
2 $ cd build
3 $ cmake .. -DCMAKE_BUILD_TYPE=Release
4 $ make

fgt comes with a CMake configuration file that lets downstream projects easily integrate fgt targets. For example, if fgt is installed to a location in the default CMake find package path, you can easily integrate fgt's C++ library into a downstream CMakeLists.txt:

1 find_package(Fgt REQUIRED)
2 add_executable(my-great-program main.cpp)
3 target_link_libraries(my-great-program Fgt::Libary-C++)

If you have installed fgt to a non-standard location, you may have to set Fgt_DIR in your downstream CMake cache.

A Note Regarding Performance

Armadillo is a template-heavy header library, so relies heavily on optimizations to achieve reasonable performance. Set CMAKE_BUILD_TYPE=Release for good results.

If you are confident in your code, you can also set ARMA_NO_DEBUG=ON to disable bounds checks on armadillo matrices. This opens up your code to problems, but if you're confident, you can get another performance boost by sidestepping those checks.
